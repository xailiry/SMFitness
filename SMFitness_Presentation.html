Актуальность и Фундаментальная Проблема

Социальная Проблема

Современная сфера любительского фитнеса характеризуется высоким порогом входа. Начинающие спортсмены сталкиваются с острым дефицитом технической методологии: они не понимают принципов периодизации, не умеют рассчитывать объем тренировочного стресса и быстро теряют мотивацию (феномен тренировочного плато).

Услуги квалифицированного тренера остаются экономически недоступными для широкого круга лиц (включая школьников и студентов).

Анализ рынка (Существующие решения)

Абсолютное большинство "бесплатных" приложений (FatSecret, MyFitnessPal) представляют собой перегруженные рекламой базы данных, не способные на динамическую адаптацию под пользователя и требующие дорогостоящей премиум-подписки для базовой аналитики.

Проектная Гипотеза

Внедрение алгоритмов машинного обучения (LLM) и сложной системы парсинга в веб-трекер позволит полностью компенсировать аналитические и базовые мотивационные функции реального тренера, обеспечивая пользователя бесплатным высококлассным сервисом.

Технологический Фундамент и Архитектура

Backend: Django & Python

Серверная логика написана на мощном фреймворке Django. Реализованы:

  • Сложная система ORM (Object-Relational Mapping).
  • Защищенная система аутентификации.
  • Контроль сессий, CSRF-защита и хэширование.

Frontend: Интерактивность и Визуализация

Отказ от тяжелых JS-фреймворков в пользу оптимизированного Vanilla JavaScript для максимальной скорости отрисовки.

Стилизация выполнена с помощью Tailwind CSS, что позволило реализовать сложные блюр-эффекты и стеклянные текстуры (Glassmorphism).

Нейросетевое Ядро: Gemini REST API

Проект использует официальную Python-библиотеку `google-generativeai`. Контекст формируется динамически: на сервер отправляется пакет данных пользователя (возраст, вес, цель, ИМТ) и история тренировок за 30 дней для комплексного анализа.

Функционал: Аналитический Журнал

Умный Парсинг Ввода (Алгоритмизация)

Пользователю не нужно тратить время на заполнение сотен полей. Разработан алгоритм, который парсит компактный формат:

Пользователь вводит: Вес 100 кг, Повторения: "10-8-6".
Система автоматически сегментирует это на 3 подхода, экстраполирует вес на каждый из них и высчитывает итоговый тоннаж тренировки еще до отправки на сервер (на уровне JS DOM).

Dynamic Formset Manager

Полностью динамическая форма добавления. Благодаря кастомным JS обработчикам, пользователь может добавлять и удалять упражнения и подходы без обновления страницы, с мгновенным пересчетом статистики базы данных.

Интерактивный Дашборд и Геймификация

Интеграция Chart.js позволяет строить графики прогресса рабочих весов и объема по мышечным группам на временной оси. Система "Стриков" отслеживает непрерывность тренировок, стимулируя дисциплину посредством психологической геймификации.

Искусственный Интеллект: AI-Компаньон

Prompt Engineering

Для стабильной работы нейросети был разработан сложный «инженерный промпт».

Скрипт запрещает модели "лить воду" и жестко структурирует ответ, заставляя ИИ анализировать именно технические аспекты прогресса (частоту кардио, дефицит нагрузок на спину).

Решена проблема кэширования: чтобы не тратить лимит API, советы генерируются не чаще раза в сутки.

Машинная генерация JSON-стратегий

Приложение не просто читает текст от ИИ. Оно запрашивает у Gemini сложную структуру данных в формате JSON.

Бэкенд расшифровывает JSON-строку и превращает ее в готовый модульный UI-план питания (с расчетом КБЖУ, жиров и белков) и недельную программу тренировок (сплит-систему).

Ответ ИИ (структурированный парсинг): { "calories": 2450, "macros": { "protein": 160, "carbs": 280 }, "plan": ["Грудь/Трицепс", "Спина/Бицепс", "Ноги"] }

Технические победы: UX/UI Дизайн

Решение проблемы "Белой Вспышки" (FOUC)

Проблема: При использовании темной темы в веб-приложениях, браузер при переходе по страницам долю секунды показывает ослепительно белый фон, прежде чем загрузится JS.

Решение: Внедрение компактного render-blocking скрипта в блок <head>, который мгновенно считывает системную тему из LocalStorage и применяет класс Dark ДО начала отрисовки дерева DOM.

Стекломорфизм (Glassmorphism)

Использование CSS свойств backdrop-filter и сложных полупрозрачных цветов палитры Slate/Zinc для создания интерфейса, который выглядит "дорого" и премиально.

Адаптивная сетка (Grid/Flexbox) гарантирует, что на мобильном устройстве (Mobile-First) интерфейс не ломается и ощущается как нативное приложение iOS/Android.
